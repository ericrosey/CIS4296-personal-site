Projects

I've worked on some neat things.
Workforce Initiative Network for Lenfest North Philadelphia Worforce Initiative

In this project, my team and I worked to create a better system for the Lenfest Collective by making a central hub that will allow their organizations to view and share their employer contact information with ease. Their current system, which distributes the information using emails, was unable to properly allow them to communicate with each other to be able to fully benefit the North Philadelphia community. While this project is currently only in the design phase, I look forward to making our ideas into a working application.

Book Store Involving a Database

In this project, I created an input aspx page for the user to put all their information which was validated on the client side to that the page did not have to do any unneccesary processing. I brought all the book information from the database in using a connection class, and placed the information into a gridview to make it easy for the user to understand. Then the user was able to pick which books they wanted along with the quantity per book and their order would be processed and shown in a serparate gridview acting as a receipt. This gridview contained a total of all the books in the order and the total price for the books selected. When the user finished the order, the database was updated with new totals for a book's sales, quantity rented, and quantity bought. These totals for the books could be seen on a management report page with the click of a button.

Bingo Game

In this project, I got to take my first shot at component-based software design. Having never done it before, it was a big learning curve in the beginning, but eventually I was able to figure it out and get a solid grasp of the concepts. I generated a bingo board with the correct numbers in the letter columns so that it would properly play out, and then had to validate the user's button clicks. If they clicked the wrong space, they would be alerted, and if they tried to skip to the next calling number while it was on their board, then they would be alerted also. Looking back, this project would be pretty simple now, but I believe it helped me a lot with struggling through problems to find the end solution.

Courses

Not as tasty as a main course, but close.

Information Systems Analysis and Design

The analysis and design phases of the System Development Life cycle are covered in detail. Methodologies for systems analysis, specifications, and design are covered. Both the Traditional Structured and Object Oriented methodologies are used by the students, working in teams, to develop real-life information systems. Emphasis is placed on well-written documentation as well as oral communication typically required during the software development life cycle. Project management tools are employed by students to monitor their progress and the costs associated with their projects. CASE tools are employed for data and information modeling and specification.

Server-Side Web Application Development

The objective of this course is to provide further depth, beyond CIS 3309, into the development of network deployed systems. Its emphasis is on distributed, multi-tier architectures. The course is divided into 3 parts. First is the theory associated with software engineering design principles and distributed processing in client/server architectures and database systems. The second part is a deeper knowledge of the ASP.NET and C#.NET languages beyond CIS 3309. This would include such topics as web and windows applications, user interfaces, event handling, database design and ADO.NET, Web Services, Ajax, user controls, and report generation. The third part of the course is a project that the student will perform in the design and programming of a distributed client/server application.

Network Architectures

This course covers the operation of computer networks and internets. It provides the background to enable students to evaluate alternative approaches to client-server computing and n-tier software development. To accomplish this the course includes communications technology, computer network technology internetworking using the TCP/IP protocol suite, client-server protocols, client-server computing, network program component models as well as issues involving security, privacy, authentication, intellectual property rights, and social changes related to computer networking.

Client-Side Scripting for the Web

Today's web applications are becoming more interactive, doing more on the client side and less on the server side. A new software design goal is to write a single code base that can support many formats, from mobile phones, to tablets, to computer monitors. This course will focus on current technologies in client side web scripting, such as: HTML, CSS, responsive web design techniques, JavaScript, JavaScript libraries (such as jQuery), JSON (or other web data formats), Ajax (a technique using JavaScript to make asynchronous calls to web APIs), and client side web application frameworks (such as AngularJS). Students will create web applications that access a database using client side code that invokes web APIs.

Component-Based Software Design

This course emphasizes component-based software development using a modern object-oriented programming (OOP) language (currently C#). Students are introduced to software development techniques applicable in a component (class)-based, integrated software development environment (IDE). Students will learn (and practice using) the OOP language, object-oriented software design techniques, and the principles of good user interface design. Students will also learn how to navigate in, and take full advantage of, an IDE in building quality software, including user interfaces to databases, sequential files, and graphics tools. Object-oriented concepts such as inheritance, polymorphism, static and dynamic binding, and interface (abstract class) components will be covered. The primary focus is on windows-based software products, but the use of ASP.NET for client-server systems development is also introduced.

A Database Management Systems

This is an introductory course in database management systems (DBMS) and file management systems. The course covers data modeling concepts, various file management techniques, data definition and manipulation using SQL, issues in data management, development and implementation of database applications, and a perspective on emerging issues in database systems. Students work in the Lab on various assignments including prototyping and SQL, utilizing state of the art DBMS and CASE tools.
